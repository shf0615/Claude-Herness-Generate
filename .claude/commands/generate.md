# 生成 Harness 工作空间

## 功能

启动多轮问答流程，与用户**共同设计**完整的 Claude Code harness 工作空间。
不限定项目类型和技术栈 — 用户自由描述需求，工具根据描述设计所有组件。

## 流程

1. 将请求路由到 `harness-expert` 代理
2. **阶段 1**：收集基础信息（项目名、描述、技术栈、输出路径）
3. **阶段 2**：与用户讨论代理角色设计（逐个确认）
4. **阶段 3**：与用户讨论命令、技能、自动化级别
5. **阶段 4**：与用户讨论工作流编排和开发方法论
6. **阶段 5**：展示完整预览，用户确认或修改
7. **阶段 6**：生成全部文件并验证

每个阶段结束后等用户回复，不会跳过任何讨论环节。

## 快速开始

```
/generate

# 或附带初始描述：
/generate 一个用 Go 写的微服务网关
```

## 生成内容

| 组件 | 说明 |
|---|---|
| `CLAUDE.md` | 项目编排指南（路由规则、工作流、质量门禁） |
| `settings.json` | 工具权限配置（根据技术栈定制） |
| `agents/*.md` | 专用代理（角色由问答决定） |
| `rules/*.md` | 编码规范（根据语言定制） |
| `commands/*.md` | 斜杠命令（功能由问答决定） |
| `hooks/hooks.json` | 自动化钩子（按自动化级别） |
| `skills/` | 项目技能（可选，由问答决定） |
