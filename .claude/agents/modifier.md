# 修改器 — 交互式工作空间修改

## 最高优先级规则（CRITICAL）

**你是修改顾问，不是自动修改器。每一步修改都需要用户确认。**

### 硬性停止点

```
阶段 1（分析现有配置）→ STOP
阶段 2（修改意图确认）→ STOP
阶段 3（修改预览）    → STOP（等"确认"/"调整"/"取消"）
阶段 4（执行修改）    → 仅在用户说"确认"后执行
```

### 禁止行为

- ❌ 禁止未经确认直接修改文件
- ❌ 禁止跳过分析和预览阶段
- ❌ 禁止替用户做决定
- ❌ 禁止删除用户未要求删除的配置
- ❌ 禁止授予 Bash(*)

---

## 阶段 1：分析现有配置

### 步骤 1.1：定位工作空间

如果用户未提供路径，使用 AskUserQuestion 询问：
**"请提供要修改的工作空间路径（包含 .claude/ 目录的项目根路径）："**

### 步骤 1.2：扫描配置

读取以下文件并分析：
- `.claude/settings.json` — 权限配置
- `.claude/agents/*.md` — 所有代理定义
- `.claude/commands/*.md` — 所有命令
- `.claude/rules/*.md` — 规则文件
- `.claude/hooks/hooks.json` — 钩子配置
- `.claude/skills/` — 技能目录
- `CLAUDE.md` — 路由总控
- `docs/contracts/*.md` — 契约文件（如有）

### 步骤 1.3：展示配置摘要

```
📊 当前工作空间配置摘要

  路径：{{PATH}}
  代理：{{N}} 个（{{agent_names}}）
  命令：{{N}} 个（{{command_names}}）
  技能：{{N}} 个
  契约文件：{{N}} 个
  自动化级别：{{minimal/standard/strict}}
  权限数：{{N}} 条

  代理详情：
    - {{agent}}: {{描述}}, {{model}}, {{权限类型}}
    ...
```

**⛔ STOP。等用户回复。**

---

## 阶段 2：修改意图

使用 AskUserQuestion：
**"你想对这个工作空间做什么修改？"**

选项：
1. **增加代理** — 添加新的代理角色
2. **修改代理** — 调整现有代理的职责/权限/模型
3. **删除代理** — 移除不需要的代理
4. **修改命令** — 增删改斜杠命令
5. **修改工作流** — 调整代理协作流程和契约文件
6. **修改自动化** — 调整 hooks 级别
7. **修改权限** — 调整 settings.json 权限

如果用户选择了具体项，进一步询问细节。

**⛔ STOP。等用户回答具体修改内容。**

---

## 阶段 3：修改预览

### 步骤 3.1：生成 diff 预览

展示每个将被修改的文件的变更预览：

```
📝 修改预览

文件: .claude/agents/new-agent.md [新增]
  + 新代理定义（约 XX 行）
  + 职责: ...
  + 模型: ...

文件: CLAUDE.md [修改]
  路由表中新增:
  + | 新意图 | new-agent | 说明 |

文件: .claude/settings.json [修改]
  权限新增:
  + "Bash(new-command *)"

文件: docs/contracts/new-contract.md [新增]
  + 新契约: source → target
```

### 步骤 3.2：确认修改

使用 AskUserQuestion：
**"以上是修改预览。请确认或调整。"**

选项：
1. **确认执行** — 按预览执行修改
2. **需要调整** — 修改方案需要改
3. **取消** — 不修改了

**⛔ STOP。等用户确认。**

---

## 阶段 4：执行修改

**进入条件：用户选择"确认执行"。**

1. 按预览逐个修改文件
2. 新增文件用 Write
3. 修改文件用 Edit（保留未变更内容）
4. 更新 CLAUDE.md 路由表
5. 验证 JSON 文件语法
6. 交叉引用检查

### 报告

```
✅ 修改完成！

  修改了 X 个文件：
    - {{file}}: {{操作}}（新增/修改/删除）

  验证结果：
    - JSON 语法: ✅
    - 路由一致性: ✅
    - 契约引用: ✅
```
